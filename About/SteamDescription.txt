[h1]DefLoadCache[/h1]
[b]Cut your modded RimWorld launch time by up to 45%.[/b]

Tired of waiting 10-15 minutes every time you launch your 500+ mod list? DefLoadCache caches the results of mod loading so you don't pay that cost every single time.

[h2]How it works[/h2]
Every time RimWorld launches, it reads thousands of XML files from every mod and then applies thousands of XML patches (compatibility fixes, balance changes, etc). On large mod lists, this takes [b]5-7 minutes[/b], and it does it from scratch every launch, even if nothing changed.

DefLoadCache saves the result after the first launch. On every launch after that with the same mod list, it loads the cached result in seconds instead of redoing all that work.

[h2]Performance[/h2]
Tested on a 576-mod Combat Extended milsim load order:
[list]
[*] Total launch time: [b]~14 min to ~8 min[/b] (45% faster)
[*] Mod XML loading + patching: [b]~6 min to ~23 seconds[/b]
[*] Cache size: ~7 MB per mod list
[/list]

Your results will vary based on mod count, hardware, and which mods you run. Larger mod lists see bigger improvements.

[h2]Features[/h2]
[list]
[*] [b]Works with any mod manager.[/b] RimPy, RimSort, or manual. DefLoadCache fingerprints whatever mods are active regardless of how they got there. Each mod list configuration gets its own cache automatically.
[*] [b]Automatic cache invalidation.[/b] The cache rebuilds automatically when you add, remove, or update mods. No manual clearing needed.
[*] [b]Multiple mod list profiles.[/b] Keeps separate caches for different mod lists. Switch between RimPy profiles without losing your cached data.
[*] [b]Per-feature toggles.[/b] Independently enable/disable "skip reading mod files" and "skip applying patches" from mod settings.
[*] [b]Safe fallback.[/b] If anything goes wrong, the game loads normally as if DefLoadCache isn't installed. No risk of save corruption.
[*] [b]Zero gameplay impact.[/b] Only affects load time. No changes to gameplay, defs, or balance.
[/list]

[h2]Safety & Self-Validation[/h2]
DefLoadCache is designed to never cause problems for other mods. If something goes wrong, it catches itself:
[list]
[*] [b]Post-load validation.[/b] On every cache hit, the mod verifies per-mod def counts against the baseline stored when the cache was built. If anything doesn't match, the bad cache is automatically deleted.
[*] [b]Self-healing.[/b] A bad cache never survives two launches. Validation failure forces a clean uncached load on the next restart.
[*] [b]One-click isolation testing.[/b] Click "Test without cache (next launch only)" in mod settings to check if an issue is cache-related. The cache is preserved and re-enables automatically.
[*] [b]Structured log output.[/b] Every launch emits a status block in the log showing cache state and validation results. Modders reviewing player logs can instantly see whether DefLoadCache was involved in a reported issue.
[/list]

[h2]For Mod Authors[/h2]
If a player files a bug report with DefLoadCache installed, search their log for "DefLoadCache Status". You'll see the cache state, validation result, and whether the cache served correct data. If there's any doubt, ask the player to click "Test without cache (next launch only)" in mod settings and reproduce the issue.

[h2]FAQ[/h2]
[b]Q: Will this break my save?[/b]
No. DefLoadCache only affects how fast the game loads. It doesn't change any defs, gameplay mechanics, or save data. You can add or remove it at any time.

[b]Q: Do I need to clear the cache when I change mods?[/b]
No. The cache automatically detects when your mod list, mod versions, or DLLs change and rebuilds from scratch. You should never need to manually clear the cache under normal use.

[b]Q: Why is the first launch still slow?[/b]
The first launch with a new mod list has to build the cache. It runs everything normally and saves the result. Every launch after that is faster.

[b]Q: What if I'm developing a mod?[/b]
Go to Options > Mod Settings > DefLoadCache and disable "Skip reading mod files" or "Skip applying patches" while you're testing changes. Or hit "Clear all cached data" to force a full rebuild.

[b]Q: Does this conflict with Performance Fish or RocketMan?[/b]
DefLoadCache targets a different part of the loading pipeline than most performance mods. It should work alongside them, but if you experience issues, please report them.

[b]Q: Where are the cache files stored?[/b]
In your RimWorld save data folder under DefLoadCache/. You can see the exact path and manage cache files from the mod settings page.

[b]Q: Is this safe to add mid-playthrough?[/b]
Yes. It only affects startup loading, not runtime gameplay. Add or remove it whenever you like.

[h2]Settings[/h2]
Access via [b]Options > Mod Settings > DefLoadCache[/b]:
[list]
[*] Enable/disable the mod entirely
[*] Toggle "skip reading mod files" and "skip applying patches" independently
[*] Set how many mod list profiles to keep cached (default 10)
[*] View cache info (size, location, last launch status)
[*] Clear all cached data
[*] Test without cache (next launch only), for one-click isolation testing
[*] Developer diagnostic dump for verifying correctness
[/list]
